Contact Book (Python)

Here to the left is a picture of one of my projects that I was making in my 10th grade year. This was a very special type of project because it was a submission for one of my AP classes so I wanted to make it as best I could. I chose to make a contact book because people use it all the time sometimes without even realizing it. The contact book stores information of people you have made note of to keep track of things like their number, name, address, etc.

Of course there wouldn't be much point of this project if you couldn't actually store any of the information anywhere so I created a file that I would append all the information into. Doing so made it way more realistic when creating a contact using this program. Of course I would still like to add tons more work into this project because I wasn't satisfied with the way it was working at the time so I added another part to see if the user would like to view any information and for which person.

This became one of the projects that took super long to do but was very enjoyable along the way, I learned many things about Python and tricks that I didn't imagine would even be legal in Python. The outcome of this project was a very great learning experience showing me how big projects can be, even though I feel like the project is still a work in progress I'm satisfied with how it is for now, and I imagine there are more projects far bigger than this one.

If you're interested in trying it out: Contact book.

Company/Service Project (C#)

This project was a rollercoaster since I wanted to make it similar to the Contact Book project. It felt so new not having to indent anything but remembering to put ; at the end of each line and being able to add more code on the same line, it was sort of overwhelming but very interesting to me.
The Company/Service Project is a project in which I wanted to have the user be presented a menu and let them have the freedom of being able to choose something from the menu. After being able to choose what they wanted I then would ask them of any changes were needed to be made in their order, after that I would make a new file and have everything they ordered added on there. This would then leave a thank you note at the end.


This screenshot to the left shows the variables made and the way I introduced the user to the program. I wanted to try my best to keep it nice and simple so the user could get through it without it being too much of a hassle.

This image to the right shows how I ended the program and the way I was able to append the text into the text file. This gives the user their own copy of the receipt without it having to go away with the program finishing.
